# Документация модулей Library Manager

Обзор архитектуры

Library Manager построен по модульному принципу с четким разделением
ответственности между компонентами.

Описание модулей

1. Модули данных (Data Layer)

models.txt

- Назначение: Определение структур данных
- Зависимости: Нет
- Используется: storage, library_core
- Функции:
  - Структура "Книга"
  - Структура "Читатель"
  - Структура "Выдача"
  - Правила валидации структур

storage.txt

- Назначение: Сохранение и загрузка данных
- Зависимости: models
- Используется: library_core
- Функции:
  - Сохранить/загрузить книги
  - Сохранить/загрузить читателей
  - Сохранить/загрузить выдачи
  - Создать резервную копию

2. Модули бизнес-логики (Business Layer)

library_core.txt

- Назначение: Основная логика библиотеки
- Зависимости: models, storage, validators
- Используется: menu, main
- Функции:
  - Инициализация библиотеки
  - Добавление/удаление книг
  - Регистрация читателей
  - Выдача/возврат книг
  - Управление каталогом

search.txt

- Назначение: Поиск книг
- Зависимости: library_core
- Используется: menu
- Функции:
  - Поиск по названию
  - Поиск по автору
  - Поиск по жанру
  - Поиск по году
  - Расширенный поиск

3. Модули интерфейса (Interface Layer)

menu.txt

- Назначение: Взаимодействие с пользователем
- Зависимости: library_core, search
- Используется: main
- Функции:
  - Главное меню
  - Подменю разделов
  - Обработка ввода
  - Подтверждения действий

4. Вспомогательные модули (Utility Layer)

validators.txt

- Назначение: Проверка данных
- Зависимости: Нет
- Используется: library_core, menu
- Функции:
  - Проверка книги
  - Проверка читателя
  - Проверка ISBN
  - Проверка телефона
  - Проверка email

Принципы взаимодействия

1. Слабая связанность

Модули взаимодействуют через четко определенные интерфейсы (функции).
Изменение внутренней реализации модуля не влияет на другие модули.

2. Направление зависимостей

Зависимости идут сверху вниз:

- Interface → Business → Data
- Никогда наоборот!

3. Независимость модулей

Модули utils/ независимы и могут использоваться любыми другими модулями.

Расширяемость

Добавление нового модуля

Чтобы добавить новый модуль:

1. Определите его назначение и слой (data/business/interface/utils)
2. Создайте файл в соответствующей папке
3. Укажите зависимости в шапке файла
4. Документируйте все функции
5. Обновите эту документацию

Пример: Добавление модуля статистики

modules/business/statistics.txt

Зависимости: library_core
Функции:
- Общая статистика библиотеки
- Самые популярные книги
- Активность читателей
- Отчеты по периодам

Тестирование модулей

Каждый модуль должен тестироваться независимо.

Папка tests/ содержит тесты для каждого модуля:

- tests/test_validators.txt
- tests/test_storage.txt
- tests/test_library_core.txt
- и т.д.

Примеры использования

Добавление книги (взаимодействие модулей)

1. Пользователь выбирает "Добавить книгу" в menu
2. menu запрашивает данные книги у пользователя
3. menu вызывает validators.ПроверитьКнигу()
4. Если данные корректны:
   - menu вызывает library_core.ДобавитьКнигу()
5. library_core создает объект через models
6. library_core сохраняет через storage.СохранитьКниги()
7. Результат возвращается в menu и выводится пользователю

Поиск книги (взаимодействие модулей)

1. Пользователь выбирает "Поиск по автору" в menu
2. menu запрашивает имя автора
3. menu вызывает search.ПоискПоАвтору(имя)
4. search обращается к library_core.ПолучитьСписокКниг()
5. search фильтрует список и возвращает результат
6. menu отображает найденные книги через display

Глоссарий

- Модуль - независимая часть программы с четкой функцией
- Слой - группа модулей одного назначения (Data, Business, Interface)
- Зависимость - использование одним модулем функций другого
- Интерфейс модуля - набор доступных функций модуля
- Связанность - степень зависимости между модулями
- Связность - степень единства внутри модуля
